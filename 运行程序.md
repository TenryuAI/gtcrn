# =============== 实时降噪 ==================================================================================
python realtime_infer.py
我已经为你编写了一个实时麦克风降噪监听程序，文件名为 `realtime_infer.py`。

这个程序使用了 `sounddevice` 库来实时获取麦克风输入，并将处理后的音频直接输出到你的耳机/扬声器中。

### 运行方法

首先，你需要确保安装了 `sounddevice` 库（如果没有安装，请先运行 `pip install sounddevice`）。

然后在终端中运行：

```bash
python realtime_infer.py
```

### 程序说明

运行后，程序会提示你选择模式：
*   **输入 `1`**：监听原始声音（不降噪）。音频会直接从麦克风传输到耳机，你可以用这个模式来确认麦克风是否正常工作，以及当前的底噪情况。
*   **输入 `2`**：监听降噪后的声音。程序会加载 GTCRN 的流式模型（`StreamGTCRN`），实时对麦克风输入进行降噪，并将纯净的声音输出到你的耳机。

**⚠️ 重要提示：**
1. **强烈建议戴上耳机进行测试！** 如果你使用电脑的外放扬声器，麦克风会把扬声器播放的声音再次录进去，产生严重的**啸叫（回音环路）**。
2. 程序的延迟非常低（每次处理 256 个采样点，在 16kHz 采样率下仅有 16 毫秒的算法延迟）。
3. 退出程序只需在终端中按下 `Ctrl + C` 即可。


# =============== 离线降噪 ==================================================================================
python infer.py

运行说明：

降噪后的文件将会保存在同一目录下，命名为 test_wavs/test_enh.wav。
请确保你的 test.wav 文件的采样率是 16000 Hz（16kHz），因为模型（GTCRN）是基于 16kHz 训练的，代码中也有 assert fs == 16000 的限制。如果采样率不对，程序会报错，你需要先将音频重采样为 16000 Hz。

我已经修改了 `infer.py` 脚本，使其现在能够自动兼容单声道和立体声（多声道）的音频文件。

修改的逻辑是：
1. 如果读取到的是立体声，它会将两个声道作为 `batch_size=2` 同时送入模型进行降噪。
2. 降噪完成后，再将两个声道合并回原来的立体声格式并保存。

你现在可以再次运行 `python infer.py`，它就能正常处理你的立体声 `test.wav` 文件了。
