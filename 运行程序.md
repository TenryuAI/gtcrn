# =============== 实时降噪 ==================================================================================
python realtime_infer.py
我已经为你编写了一个实时麦克风降噪监听程序，文件名为 `realtime_infer.py`。

这个程序使用了 `sounddevice` 库来实时获取麦克风输入，并将处理后的音频直接输出到你的耳机/扬声器中。

### 运行方法

首先，你需要确保安装了 `sounddevice` 库（如果没有安装，请先运行 `pip install sounddevice`）。

然后在终端中运行：

```bash
python realtime_infer.py
```

### 程序说明

运行后，程序会提示你选择模式：
*   **输入 `1`**：监听原始声音（不降噪）。音频会直接从麦克风传输到耳机，你可以用这个模式来确认麦克风是否正常工作，以及当前的底噪情况。
*   **输入 `2`**：监听降噪后的声音。程序会加载 GTCRN 的流式模型（`StreamGTCRN`），实时对麦克风输入进行降噪，并将纯净的声音输出到你的耳机。

**⚠️ 重要提示：**
1. **强烈建议戴上耳机进行测试！** 如果你使用电脑的外放扬声器，麦克风会把扬声器播放的声音再次录进去，产生严重的**啸叫（回音环路）**。
2. 程序的延迟非常低（每次处理 256 个采样点，在 16kHz 采样率下仅有 16 毫秒的算法延迟）。
3. 退出程序只需在终端中按下 `Ctrl + C` 即可。

终于解决了！ 板载耳机口重启后持续作为默认输出设备，USB 麦克风保持为默认录音设备。

根本原因总结：

module-switch-on-port-available 这个 PulseAudio 模块会在检测到音频端口"变为可用"时自动抢占默认设备。USB 设备在每次启动时端口都会经历一个"不可用 → 可用"的初始化过程，导致它每次都自动覆盖了我们设置的 rk809 默认值。

最终生效的设置：

配置文件	修改内容
/etc/pulse/default.pa	注释掉 module-switch-on-port-available（防止 USB 抢占） + 加了 set-default-sink rk809
/etc/rc.local	启动时执行 amixer ... Playback Path HP（确保 3.5mm 耳机路径生效）
现在所有程序（aplay、gtcrn_realtime 等）都会通过板载 3.5mm 耳机口输出，USB 麦克风用于录音，重启后永久有效。

```bash
# 设置 USB 声卡为默认播放（输出）
pactl set-default-sink alsa_output.usb-KUAIYU_ELECTRONIC_KUAIYU_USB_MIC-00.analog-stereo

# 设置 USB 声卡为默认录音（输入）
pactl set-default-source alsa_input.usb-KUAIYU_ELECTRONIC_KUAIYU_USB_MIC-00.analog-stereo

# 验证
pactl info | grep -E "Default Sink|Default Source"
```
